## 1. Purpose

To ensure all guest name change requests are evaluated and handled correctly based on the hotel country policy, with clear communication to agents.

## 2. Scope

This procedure applies to all reservations requiring a guest name change, processed by the front desk or booking management team.

## 3. Responsibilities

- **Front Desk / Booking Team**: Receive name change request, identify hotel country, evaluate eligibility, communicate with hotel (if applicable), and inform the agent of the outcome.
- **Hotel Operations (Other Countries)**: Respond to name change inquiries from the booking team.

## 4. Procedure

### Step 1 — Receive & Verify Request

- Receive name change request from Agent (AGT).
- Check booking details in the system (guest name, hotel, check-in date, hotel country).

### Step 2 — Identify Hotel Country

#### Case A: Hotel is in **Japan or Korea**

- Check reason for name change request.
- **Wrong typing / spelling error?**
  - ✅ **YES** — Correct the name in the system → Inform agent → Close the booking.
  - ❌ **NO** — Inform agent the request is **rejected**. Advise that Japan/Korea hotels only allow corrections for wrong typing → Close the booking.

#### Case B: Hotel is in **Other Countries**

- Contact hotel to check if name change is allowed.
- **Hotel approves?**
  - ✅ **YES** — Update the guest name in the system → Inform agent → Close the booking.
  - ❌ **NO** — Inform agent the name change was rejected by the hotel → Close the booking.

## 5. Notes

- **Japan & Korea hotels**: Name change is **ONLY** allowed for wrong typing / spelling errors. All other reasons must be rejected.
- **Other countries**: Always check with the hotel before confirming or rejecting the request.
- Document all communications with hotels and agents in the booking system.
- Ensure the agent is informed promptly to avoid delays in guest preparation.
