## 1. Purpose

To ensure all special requests from agents are categorized and handled efficiently. Simple preference-based requests are auto-confirmed to agents subject to hotel availability, while special arrangement requests require verification with the hotel before the response is forwarded to the agent.

## 2. Scope

This procedure applies to all special requests submitted by agents on behalf of guests for active bookings, processed by the front desk or booking management team.

## 3. Responsibilities

- **Front Desk / Booking Team**: Receive the special request, identify the request type, auto-confirm Type A requests, contact hotel for Type B requests, and communicate the outcome to the agent.
- **Hotel Operations**: Review and respond to Type B special arrangement requests forwarded by VNOP.

## 4. Procedure

### Step 1 — Receive & Verify Request

- Receive the special request from Agent (AGT).
- Check booking details in the system (guest name, hotel, check-in / check-out dates, booking status).
- Identify and categorize the request type — refer to the **Request Type Reference Table** below.

### Step 2 — Handle Based on Request Type

#### Type A — **Simple Request (Preference-Based)**

- 예시: high floor, good view, nice room, quiet room, near elevator, non-smoking preference.
- **VNOP does NOT need to contact the hotel.**
- Auto-confirm to Agent with the **standard response**:

> "We have noted your request and forwarded it to the hotel. Please be advised that all requests are subject to hotel availability at the time of check-in."

- Log the request and agent confirmation in the booking system.
- Close the request. ✔ Done.

#### Type B — **Special Arrangement Request**

- 예시: airport pick-up, extra bed, baby cot, delivery service, early check-in, late check-out, special meals.
- Contact the hotel to check availability and confirm whether the request can be fulfilled.
- **Hotel confirms the request**: Forward confirmation details to the agent (including any extra charges) → Log → Close. ✔
- **Hotel cannot fulfill**: Inform Agent → Suggest alternatives if available → Log → Close.
- **No response from hotel**: Follow up. If still no response, escalate to supervisor. Keep agent informed.

## 5. Notes

- **Type A** requests must always be communicated with the standard availability disclaimer — **never guarantee fulfillment** to the agent.
- **Type B** requests must always be verified with the hotel before confirming to the agent — **do not auto-confirm**.
- If a request falls between Type A and Type B and it is unclear, **treat it as Type B** and check with the hotel.
- Always log the request type, action taken, hotel response (if applicable), and agent notification in the booking system.
- If the hotel charges extra for a Type B request, clearly communicate the cost to the agent before confirming.
- Agents must be informed promptly at every stage to ensure smooth guest preparation.

## Request Type Reference Table

| Type A — **Simple Requests** (자동 컨펌) | Type B — **Special Arrangements** (호텔 확인 필요) |
|---|---|
| High floor room | Airport pick-up / transfer |
| Good view / sea view / city view | Extra bed / baby cot |
| Nice / quiet room | Delivery service (flowers, cake, gifts) |
| Near elevator / specific room location | Early check-in / late check-out |
| Non-smoking room preference | Interconnecting / adjoining rooms |
| Honeymoon / anniversary setup (standard) | Special dietary meal arrangement |
